Goal

We aim to make a minesweeper game in OCaml that is playable in the terminal. The game can be installed off of our github.

Process

We implement the game game logic in a purely fuctional manner and implement unit and integration tests for each component of the game to ensure proper functionality.

Highlights

The game is developed in a purely functional manner, meaning that no mutable states are used and each function only depends on its input.

Overview


This was the final project for my functional programming project in undergrad. The game was developed in OCaml and was tested using ounit tests. OCaml is the functional language that the class was taught in. It belongs to the ML (MetaLanguage) family of functional programming languages and is known for its expressiveness, strong static typing, and powerful type inference, making it suitable for a wide range of applications. OCaml does have the ability to implement mutable states, but we chose to avoid this in our implementation and instead use a purely functional approach.
